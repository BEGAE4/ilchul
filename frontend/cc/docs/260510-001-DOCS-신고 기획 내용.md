## 신고 기준 구체화

1. 신고의 범위

1.1 신고 대상 : 유저, 플랜, 댓글
1.2 신고 사유 : 

- 플랜 : 스팸/광고, 허위정보, 음란성, 욕설/비방, 저작권 침해, 기타(직접 입력)
- 유저 : 저작권 침해, 부적절한 닉네임/프로필, 타인 사칭, 기타(직접 입력)

1.3 신고 자격 조건

- 본인이 본인 신고 불가능
- 로그인 유저만 신고 가능

1. 신고의 처리

2.1 플랜
visibility(유저 제어) / 운영 제어 분리

자동 블라인드 임계치 : 서로 다른 신고자 N명이 신고할 경우 자동 블라인드

- 블라인드 중 작성자 권한 제한 : 수정/삭제 불가
- 처리 결과 분기 : 관리자 판단에 의해서 삭제/복원
- 캐시/검색 동기화

2.2 유저

- 중복 신고 제한 : 동일한 신고자 + 항목-> 동일한 피신고자 (횟수 제한)
    - 제재 종류 및 권한 범위 : 경고, 횟수에 따른 7일 누적식 정지, 최대 횟수(5회 - 35일) 초과 후 영구 정지

1. 사후 처리

3.1 통보 (신고자 / 피신고자)

앱 서비스 내 알림 방식 

경고 처리 ( 경고 통보 후, 일주일 이내 조치하지 않을 경우 정지처리) 

경고 조치와는 무관하게, 경고 횟수에 따라 정지 기간이 누적되는 방식으로 진행. 
최대 5회. 5회 이상 시 영구 정지.  

3.2 이의제기 프로세스

- 일단 이메일로 진행. (추후, 이의제기 가능 시점, 이의제기 횟수 제한 기능 등 필요 시 고도화)
